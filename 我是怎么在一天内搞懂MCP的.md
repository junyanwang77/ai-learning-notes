# 我是怎么在一天内搞懂 MCP 的

> 实践笔记 · 2026-06-04

---

## 一早的恐慌

早上看到李笑来又出了一本新书，速度惊人。

顺手翻了翻他的 GitHub 仓库，满眼都是没见过的项目和工具，一股熟悉的"知识恐慌"扑面而来——要学的东西实在太多了。

但与其焦虑，不如直接上手。

## 今天的目标：搞懂 MCP 怎么用

我注意到仓库里有一个叫 **Codex Octopus** 的项目，看不太懂，就把链接扔给了 Claude，让它一步步带我走。

操作下来，才明白：**Codex Octopus 其实是一个 MCP 工具**，专门用来在 Claude 桌面端调用 Codex。

MCP（Model Context Protocol）说白了，就是让 AI 能够直接连接并操作外部工具和文件的一套协议。今天算是对它有了第一手的感觉，但要真正用熟，还需要更多练习。

## 顺手做了一个实验：VMark MCP

今天还测试了另一个 MCP 工具——**VMark MCP**。

结果挺有意思：通过它，Claude 可以直接读取和修改我在 VMark 里打开的文档，不需要复制粘贴，不需要手动操作，AI 直接在编辑器里帮你写。

**这篇文章本身，就是 Claude 通过 VMark MCP 整理并写入的。**

## 小结

今天最大的收获不是学会了某个具体工具，而是对 MCP 的运作方式有了真实的感受——它让 AI 从"对话框里的助手"变成了"能动手的协作者"。

接下来要做的，就是多练习，多理解不同项目的结构。
